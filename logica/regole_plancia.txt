# Regole plancia (legacy)

Documento storico delle convenzioni Lovelace. Usare solo come riferimento di continuità visiva: tutte le indicazioni attuali sono riportate nei file core.

## Struttura & Layout
- Mobile-first con `sections` e `max_columns` per un ordine coerente.
- Tre aree fisse: **Stato & Comandi rapidi**, **KPI**, **Trend/Lock/Diagnostica**.
- Card obbligatorie: glance/chip di stato, pulsanti rapidi, markdown “Come decide” (ora rimandato al core + file logico di modulo).

## Palette
- Rosso = esterno; giallo = zona giorno; blu = zona notte; verde = bagno; neutri = derivate/logiche speciali.
- Tile/grafici senza riempimento, una metrica per card, soglie come linee nominate.

## Debug & manutenzione
- Sezione finale per sensori grezzi e diagnostica (entity-filter su `unknown/unavailable`).
- Badge per lock/override manuali.

## Note sulle logiche
Le descrizioni dettagliate di VMC, AC, heating e ventilazione sono state migrate in `core/regole_core_logiche.md` e nei rispettivi file logici (`1_vent.txt`, `2_vmc.txt`, `3_heating.txt`, `4_ac.txt`, `6_surplus.txt`). Qui restano solo indicazioni generiche di layout.
